JS易记错知识梳理

发表于 2017-10-20 | 分类于 js

在杭州待了一个星期，没有面试，笔试也少的可怜。总的来说还是因为自己基础知识太薄弱，再加上React,React Native等
常用的框架都不会，早工作会很被动。
根据面试，笔试的情况我发现

  1. js基础部分没有记得很清楚，对于方法如何传入参数，返回什么值都记不清楚
  2. jQuery 中相当一部分都没有仔细的去看
  3. 网络知识需要重新梳理
    4、css 对flex 布局 定位，css选择器等没有全部了解，但是这是重要考察知识点
    针对这些问题，这篇文章着重列出js部分不容易掌握的一些知识点

    第一章：数据类型

    1. null表示一个空对象指针
      typeof null===’object’
    2. 能转换为false的值有：’’(空字符串)，0，NaN,null undefined。其他值都转换为true
    3. 浮点数值需要注意
      (0.1+0.2==0.3)=false
      (0.15+0.25==0.4)=true
    4. Number()方法数值转换：
      //null->0,undefined->NaN,’’->0
    5. null, undefined 没有toString()方法，只有String()方法
    6. 函数没有重载(一个函数两个定义)
    7. 执行环境定义了变量或函数有权访问的其他数据，决定了他们各自的行为。执行环境中的变量对象就是当前执行代码中的变量和函数的集合，
      作用域链表示当前执行环境和外部执行环境的有序访问的向上搜索的链表

      第二章：引用类型

    8. 数组方法中不影响原数组的有concat()，slice()，forEach(),map()，filter();splice()返回删除的项
    9. Data类型的方法

      第三章：对象

    10. 创建对象的方法：工厂模式，构造函数模式，原型模式，组合模式，动态原型模式，寄生构造函数模式，
    11. 实现继承的方法： 原型链，借用构造函数，组合继承，寄生组合式继承
